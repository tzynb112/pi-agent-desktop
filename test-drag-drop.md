# 拖拽上传功能测试

## 测试步骤

1. **启动应用**
   ```bash
   npm run start
   ```

2. **测试拖拽单个文件**
   - 在文件管理器中选择一个文件
   - 拖拽到输入框区域
   - 观察是否显示紫色虚线边框
   - 松开鼠标，检查文件是否添加到附件列表

3. **测试拖拽多个文件**
   - 选择多个文件
   - 拖拽到输入框区域
   - 检查所有文件是否都添加到附件列表

4. **测试拖拽文件夹**
   - 选择一个文件夹
   - 拖拽到输入框区域
   - 检查是否提示不支持文件夹

5. **测试拖拽取消**
   - 拖拽文件到输入框区域
   - 将文件拖出区域
   - 检查紫色边框是否消失

## 预期结果

1. **拖拽时**
   - 显示紫色虚线边框
   - 显示上传图标
   - 显示"拖拽文件到这里"文字

2. **放下文件后**
   - 文件添加到附件列表
   - 显示文件名、类型图标、大小
   - 可以删除附件

3. **发送消息**
   - 附件随消息一起发送
   - AI 能够读取附件内容

## 故障排除

### 拖拽无反应
- 检查是否拖拽到正确的区域
- 确认文件格式是否支持
- 查看控制台是否有错误

### 文件无法读取
- 检查文件权限
- 确认文件编码格式
- 查看 API 错误信息

### 拖拽卡顿
- 检查文件大小
- 减少同时拖拽的文件数量
- 查看系统资源占用

## 支持的拖拽方式

1. **从文件管理器拖拽**
   - Windows 资源管理器
   - macOS Finder
   - Linux 文件管理器

2. **从其他应用拖拽**
   - 浏览器中的图片
   - 文本编辑器中的文件
   - 压缩软件中的文件

3. **从桌面拖拽**
   - 直接从桌面拖拽文件
   - 支持快捷方式

## 注意事项

1. **文件大小限制**
   - 单个文件最大 10MB
   - 总附件大小最大 50MB

2. **文件格式**
   - 支持代码文件、图片、文档
   - 不支持可执行文件
   - 不支持系统文件

3. **安全提示**
   - 不要拖拽敏感文件
   - 检查文件内容再发送
   - 注意文件来源
