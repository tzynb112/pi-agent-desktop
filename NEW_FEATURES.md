# 新功能说明

本文档介绍了最近添加的三个新功能：提示模板系统、包管理系统和 /goal 多代理长时间目标开发功能。

## 1. 提示模板系统 (Prompt Templates)

### 功能说明

提示模板系统允许用户创建可复用的提示模板，通过输入 `/模板名` 快速调用。

### 使用方法

1. **创建模板**：
   - 打开设置 → 提示模板
   - 点击"新建模板"
   - 填写模板名称、描述和内容
   - 保存模板

2. **使用模板**：
   - 在输入框中输入 `/` 触发自动补全
   - 选择或输入模板名称
   - 按 Tab 或 Enter 选择模板
   - 模板内容会自动展开

3. **模板参数**：
   - `$1`, `$2`, ... - 位置参数
   - `$@` 或 `$ARGUMENTS` - 所有参数
   - `${@:N}` - 从第 N 个参数开始
   - `${@:N:L}` - 从第 N 个参数开始的 L 个参数

### 示例模板

```markdown
---
description: 代码审查模板
argument-hint: "<file-path>"
---
请审查以下文件的代码：$1

关注点：
- 代码质量
- 性能问题
- 安全漏洞
- 最佳实践
```

使用：`/review src/main.ts`

### 存储位置

- 全局模板：`~/.pi/agent/prompts/*.md`
- 项目模板：`.pi/prompts/*.md`

## 2. 包管理系统 (Pi Packages)

### 功能说明

包管理系统允许用户安装、卸载和管理 pi 包，包含扩展、技能、提示模板和主题。

### 使用方法

1. **安装包**：
   - 打开设置 → 包管理器
   - 点击"安装包"
   - 输入包来源（npm、git 或本地路径）
   - 选择安装范围（全局或项目）
   - 点击"安装"

2. **包来源格式**：
   - npm: `npm:@scope/package@version`
   - Git: `git:github.com/user/repo@tag`
   - 本地: `/path/to/package`

3. **管理包**：
   - 启用/禁用包
   - 卸载包
   - 查看包资源（扩展、技能、模板、主题）

### 示例命令

```bash
# 安装 npm 包
pi install npm:@pi-community/tools

# 安装 Git 包
pi install git:github.com/user/pi-package

# 安装本地包
pi install /path/to/package
```

## 3. /goal 多代理长时间目标开发功能

### 功能说明

/goal 命令允许用户设定一个长期目标，系统会自动分解目标为多个子任务，并使用多个代理并行执行。

### 使用方法

1. **启动目标**：
   - 在输入框中输入 `/goal 目标描述`
   - 或点击输入框旁的目标按钮

2. **执行过程**：
   - 系统自动分析目标并制定执行计划
   - 显示子任务列表和预计时间
   - 多个代理并行执行子任务
   - 实时显示进度和状态

3. **查看状态**：
   - 目标进度条
   - 子任务状态
   - 代理状态
   - 执行结果

### 示例目标

```
/goal 创建一个完整的用户认证系统，包括注册、登录、密码重置功能
```

### 功能特点

- **智能规划**：自动分析目标并制定执行计划
- **并行执行**：多个代理同时执行独立子任务
- **依赖管理**：自动处理子任务之间的依赖关系
- **实时反馈**：实时显示执行进度和状态
- **错误处理**：自动重试失败的子任务

### 代理配置

- 默认最大并发代理数：3
- 代理能力：read、write、edit、bash
- 代理状态：idle、busy

## 注意事项

1. **安全性**：
   - 包管理系统具有完全系统访问权限
   - 安装第三方包前请审查源代码
   - 使用沙箱模式保护系统安全

2. **性能**：
   - 多代理执行会消耗更多系统资源
   - 建议根据系统性能调整最大并发代理数
   - 长时间目标可能需要较长时间完成

3. **兼容性**：
   - 提示模板兼容 pi-coding-agent 格式
   - 包格式兼容 pi-coding-agent 规范
   - 目标执行功能为新增功能
