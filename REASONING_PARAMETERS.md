# 模型推理参数说明

## 概述

PianoAgent 现在支持完整的模型推理参数调节，让你可以精确控制 AI 的推理行为。

## 参数列表

### 1. 采样温度 (Temperature)

**作用**: 控制输出的随机性

| 值 | 效果 | 适用场景 |
|---|------|----------|
| 0.0 | 确定性输出 | 代码生成、数学计算 |
| 0.3 | 低随机性 | 技术文档、精确回答 |
| 0.7 | 平衡 | 通用对话、创意写作 |
| 1.0 | 中等随机性 | 头脑风暴、创意生成 |
| 1.5+ | 高随机性 | 艺术创作、探索性任务 |

**建议值**:
- 代码任务: 0.0 - 0.3
- 通用任务: 0.7
- 创意任务: 0.9 - 1.2

### 2. Top P (核采样)

**作用**: 控制词汇选择的多样性

| 值 | 效果 | 适用场景 |
|---|------|----------|
| 0.1 | 高度聚焦 | 精确回答、代码生成 |
| 0.5 | 中等聚焦 | 技术文档 |
| 0.9 | 平衡 | 通用任务 |
| 1.0 | 完全多样 | 创意写作 |

**建议值**:
- 代码任务: 0.1 - 0.3
- 通用任务: 0.9
- 创意任务: 0.95 - 1.0

**注意**: Temperature 和 Top P 通常只调整一个，不要同时使用极端值。

### 3. 推理强度 (Reasoning Effort)

**作用**: 控制模型思考的深度

| 值 | 效果 | 适用场景 |
|---|------|----------|
| 快速 (low) | 减少思考时间，快速响应 | 简单问题、快速查询 |
| 平衡 (medium) | 平衡速度与质量 | 通用任务 |
| 深度 (high) | 深度思考，更高质量 | 复杂问题、代码审查 |

**建议值**:
- 简单任务: 快速
- 通用任务: 平衡
- 复杂任务: 深度

**注意**: 此参数仅对支持的模型有效（如 OpenAI o1、o3 系列）。

### 4. 频率惩罚 (Frequency Penalty)

**作用**: 减少重复词汇的出现

| 值 | 效果 | 适用场景 |
|---|------|----------|
| -2.0 | 鼓励重复 | 强调关键点 |
| 0.0 | 无惩罚 | 通用任务 |
| 1.0 | 中等惩罚 | 长文本生成 |
| 2.0 | 强惩罚 | 避免重复 |

**建议值**:
- 代码任务: 0.0
- 长文本: 0.5 - 1.0
- 创意写作: 0.3 - 0.7

### 5. 存在惩罚 (Presence Penalty)

**作用**: 鼓励引入新话题

| 值 | 效果 | 适用场景 |
|---|------|----------|
| -2.0 | 聚焦当前话题 | 深入讨论 |
| 0.0 | 无惩罚 | 通用任务 |
| 1.0 | 中等鼓励 | 头脑风暴 |
| 2.0 | 强鼓励 | 探索新想法 |

**建议值**:
- 深入讨论: -0.5 - 0.0
- 通用任务: 0.0
- 头脑风暴: 0.5 - 1.0

### 6. 最大生成数 (Max Tokens)

**作用**: 限制输出长度

| 值 | 效果 | 适用场景 |
|---|------|----------|
| 1024 | 短回答 | 快速查询 |
| 2048 | 中等长度 | 一般对话 |
| 4096 | 标准长度 | 代码生成 |
| 8192 | 长回答 | 文档生成 |
| 16384+ | 超长回答 | 复杂任务 |

**建议值**:
- 简单任务: 1024 - 2048
- 代码任务: 4096 - 8192
- 文档任务: 8192 - 16384

## 预设配置

### 代码生成模式
```
Temperature: 0.0
Top P: 0.1
Frequency Penalty: 0.0
Presence Penalty: 0.0
Max Tokens: 4096
```

### 通用对话模式
```
Temperature: 0.7
Top P: 0.9
Frequency Penalty: 0.0
Presence Penalty: 0.0
Max Tokens: 4096
```

### 创意写作模式
```
Temperature: 1.0
Top P: 0.95
Frequency Penalty: 0.5
Presence Penalty: 0.5
Max Tokens: 8192
```

### 深度思考模式
```
Temperature: 0.3
Top P: 0.5
Reasoning Effort: high
Max Tokens: 8192
```

## 使用建议

1. **代码任务**
   - 使用低温度 (0.0 - 0.3)
   - 使用低 Top P (0.1 - 0.3)
   - 使用标准或更高的 Max Tokens

2. **通用任务**
   - 使用默认设置
   - 根据需要微调

3. **创意任务**
   - 使用较高温度 (0.7 - 1.0)
   - 使用较高 Top P (0.9 - 1.0)
   - 适当使用频率惩罚

4. **复杂任务**
   - 使用深度推理强度
   - 使用较低温度
   - 使用较高的 Max Tokens

## 注意事项

1. **参数冲突**
   - Temperature 和 Top P 通常只调整一个
   - 不要同时使用极端值

2. **模型兼容性**
   - 不是所有模型都支持所有参数
   - Reasoning Effort 仅对特定模型有效

3. **性能影响**
   - 较高的 Max Tokens 会增加响应时间
   - 深度推理会增加计算成本

4. **实验性**
   - 不同任务可能需要不同的配置
   - 建议记录有效的配置组合
